# Алгоритм работы программы «Модуль оценки АРВ»

Документ описывает порядок работы приложения на основе кода (`Application.cs`, `MainWindow.cs`, `compModel`) и руководства пользователя.

---

## 1. Запуск приложения

1. **Точка входа** — `Application.Main()`:
   - создаётся экземпляр WPF-приложения;
   - создаётся и показывается главное окно `MainWindow`;
   - запускается цикл сообщений `app.Run()`.

2. **Инициализация главного окна** (`MainWindow`):
   - создаётся модель данных `compModel` (`fInf`);
   - загружается UI из `mainwindow.baml` (интерфейс с блоками файлов, чекбоксами, кнопкой «Расчёт», прогресс-баром).

3. **При загрузке окна** (`Window_Loaded`):
   - в каталог программы при необходимости записываются EPPlus.dll, OxyPlot.dll, OxyPlot.Wpf.dll (из ресурсов);
   - при ошибке записи выводится сообщение с контактами и возможно завершение приложения.

---

## 2. Подготовка данных (ввод файлов)

Пользователь задаёт входные данные **перетаскиванием файлов** (Drag & Drop) на окно.

1. **Обработка Drop** (`files_drop`):
   - из перетаскиваемых путей формируются списки в `compModel`:
     - **Расчётные режимы** — файлы `*.rst`;
     - **Траектории утяжеления** — файлы `*.ut2`;
     - **Контролируемые сечения** — один файл `*.sch`;
     - **Тестовое возмущение** — один файл `*.scn`;
     - **Расчётное задание** — один файл `*.xlsx` (или `*.xls`).
   - вызываются:
     - `rgms_add_to_main_win(fInf.regims)` — отображение списка режимов в UI;
     - `ut2_add_to_main_win(fInf.ut2s)` — список траекторий утяжеления;
     - `sch_add_to_main_win(fInf.schs)` — файл сечений;
   - в поля «Тестовое возмущение» и «Расчётное задание» выводятся имена файлов (без расширения) для `scn` и xlsx.

2. **Разбор файла задания** (при обращении к `fInf.xlxs` в `compModel`):
   - из xlsx читаются листы: **rgms**, **rems**, **ut**, **arv**, **grf**;
   - заполняется структура `comInfo`: базовые режимы, ремонтные схемы, траектории утяжеления, настройки АРВ, регистрируемые параметры для графиков;
   - имена файлов из перетаскивания сопоставляются с записями на листах (по именам без расширения).

3. **Двойной клик** по спискам режимов, траекторий или сечений очищает соответствующий список (обработчики в `Connect`).

## 3. Нажатие кнопки «Расчёт» (`start_Click`)

### 3.1. Подготовка к расчёту

1. **Проверка лицензии** — `license_check()` (в текущей сборке тело пустое — обход).
2. Кнопка «Расчёт» блокируется, курсор переключается в «ожидание».
3. **Диалог** — «Начать расчет?»; при отказе дальнейший расчёт не выполняется.

### 3.2. Предварительная настройка режимов (до цикла)

Для каждого **базового режима** из `fInf.xlxs.rgms` (у которого задано имя файла `sName`):

1. Создаётся объект Rastr (ПК RUSTab) через COM.
2. Загружаются:
   - сценарий возмущения `fInf.scns` в шаблон сценария;
   - файл режима `item.sName` в шаблон «динамика.rst».
3. В таблице `com_dynamics` задаются параметры расчёта (MaxResultFiles, SnapMaxCount, SnapAutoLoad, ResultFlowDirection, Tras и т.д.).
4. При необходимости увеличивается время расчёта (Tras) относительно времени старта возмущения (TimeStart/Delay).
5. Режим сохраняется: `rastr.Save(item.sName, text)`.

### 3.3. Создание выходных документов

- Создаётся экземпляр **Microsoft Word** и новый документ (для графического отчёта).
- Создаётся **ExcelPackage** (EPPlus), добавляется лист с именем из файла задания; формируется шапка таблицы:
  - «Схема сети», «Регистрируемый параметр», «Степень демпфирования», «Результат проверки» (подстолбцы: «Демпфирование переходного процесса», «Эффективность PSS»), «Примечание».
- Задаётся максимальное значение прогресс-бара (по числу режимов и ремонтных схем).

---

## 4. Расчётный цикл

Цикл выполняется по **каждому базовому режиму** из `fInf.xlxs.rgms` (с непустым `sName`).

### 4.1. Подготовка к режиму

- Создаётся (или переиспользуется) папка с именем режима в каталоге программы; при наличии — очищаются файлы в ней.
- В таблицу Excel добавляется строка-заголовок с именем режима.
- Загружается режим в Rastr: `rastr2.Load(..., closure._0024VB_0024Local_rg.sName, text2)` (шаблон «динамика.rst»).

### 4.2. Проверка сходимости режима (нормальная схема)

- Выполняется расчёт установившегося режима: `rastr2.rgm("")`.
- **Если режим не балансируется** (`rgm != AST_OK`):
  - в Excel записывается «Режим не балансируется в нормальной схеме»;
  - прогресс-бар увеличивается на число ремонтных схем этого режима + 1;
  - переход к следующему базовому режиму.
- **Если режим балансируется** — выполняется оценка для нормальной схемы (см. ниже).

### 4.3. Оценка для нормальной схемы

1. Вызов **`get_pss_quality(rastr2, excelWorksheet, ...)`** для схемы «Нормальная схема»:
   - внутри вызывается **`DynCalc(r)`** — расчёт динамики с PSS включённым и выключенным (см. п. 5);
   - по результатам вычисляются степень демпфирования, вывод о демпфировании переходного процесса и эффективности PSS;
   - строки записываются в Excel; при включённой опции «Сохранять графики» формируются графики и вставка в Word (`get_grf`).
2. Обновление прогресс-бара.
3. При опции «Сохранять режимы» — сохранение `.rst` в папку режима.

### 4.4. Обработка ремонтных схем

Для каждой **ремонтной схемы** из `fInf.xlxs.rems`, привязанной к текущему базовому режиму (`rId` содержит `rem.id`):

1. В Rastr загружается тот же базовый режим; в таблицах `vetv`/`node` выставляется отключение элементов по `rem.elements` (sta = 1 для выведенных ветвей/узлов).
2. Повторно вызывается `rastr2.rgm("")` — проверка сходимости ремонтной схемы.
3. **Если заданы сечение, МДП и траектория** (`rem.schId != -1`, `rem.mpdValue != -1`, `rem.utId != -1`):
   - вызывается **`get_mpd(rastr2, rem.schId, rem.mpdValue, rem.utId)`** — ограничение перетоков по траектории утяжеления до заданного МДП по сечению;
   - затем выполняется `get_pss_quality(...)` для этой ремонтной схемы.
4. **Если режим не балансируется** и при этом параметры ремонтной схемы неполные — в Excel пишется «Режим не балансируется в ремонтной схеме».
5. При «Сохранять режимы» — сохранение `.rst` ремонтной схемы в папку режима.
6. Обновление прогресс-бара.

### 4.5. Завершение цикла по режимам

- Оформление границ таблицы в Excel.
- Сохранение Excel: **«Приложение. Результат в табличном виде.xlsx»** в каталог программы.
- Оформление и сохранение документа Word: **«Приложение. Результат в графическом виде.docx»**.
- Закрытие Word и Excel.
- Сообщение пользователю о завершении расчёта и путях к файлам.
- Восстановление кнопки «Расчёт» и курсора.

При любой исключительной ситуации в блоке расчёта сохраняются уже сформированные xlsx и docx, выводится сообщение об ошибке с контактами; при подтверждении возможно завершение приложения.

## 5. Расчёт динамики и оценка АРВ

### 5.1. DynCalc(Rastr r)

Используется шаблон «автоматика.dfw» и сценарий `fInf.scns`.

1. **Расчёт с включённым PSS:**
   - загрузка сценария, запуск `r.FWDynamic().Run()`;
   - для каждого регистрируемого параметра из `fInf.xlxs.grf`: чтение трасс из Rastr (`GetChainedGraphSnapshot` по таблице/параметру и id), формирование временных рядов (время t, значение v);
   - результаты сохраняются в структурах `grfData` с полями `name`, `pssOn`, `sPoint` (момент возмущения).

2. **Изменение настроек АРВ** (по листу arv): в таблицах Rastr задаются значения параметров АРВ для варианта «PSS выведен».

3. **Расчёт с выключенным PSS:**
   - повторная загрузка сценария и `FWDynamic().Run()`;
   - те же трассы собираются в `pssOff` для каждого параметра из grf;
   - `pssOff` записывается в соответствующий элемент списка `grfData`.

4. Возвращается список `grfData` (по одному элементу на каждый регистрируемый параметр).

### 5.2. get_pss_quality(...)

По результатам `DynCalc` для каждой записи из `fInf.xlxs.grf`, для которой есть данные в списке:

1. **Степень демпфирования** — отношение размаха колебаний после возмущения (например, после sPoint+15 с) к размаху в момент возмущения; запись в столбец «Степень демпфирования».
2. **Демпфирование переходного процесса:**
   - если степень демпфирования &gt; 0.01 — анализ экстремумов и частоты (0.6–2 Гц); при попадании в диапазон — «Не демпфируется», иначе «Демпфируется*»;
   - иначе — «Демпфируется».
3. **Эффективность PSS** — сравнение взвешенных по времени квадратов отклонений от установившегося значения для трасс «PSS введён» и «PSS выведен» в заданном интервале; результат: «Эффективен» или «Не эффективен».
4. При включённой опции «Сохранять графики» — вызов **`get_grf(...)`**: построение графиков (OxyPlot), сохранение в PNG, вставка в документ Word с подписью.

### 5.3. get_mpd(Rastr r, сечение s, значение val, траектория ut)

Ограничение перетоков по траектории утяжеления до заданного МДП по сечению:

1. Загрузка траектории утяжеления из xlxs по `ut`, шаблон «траектория утяжеления.ut2».
2. Прогон траектории до конца: `rgm("")`, затем цикл `step_ut("z")` до завершения.
3. Загрузка файла сечений `fInf.schs` (шаблон «сечения.sch») и повторная загрузка траектории.
4. В таблице `sechen` выбирается сечение `ns = s`, читается текущее значение перетока по сечению (`psech`).
5. Если переток больше заданного `val`, в цикле выполняется подбор коэффициента в `ut_common.kfc` и шаги траектории до сближения перетока по сечению с `val` (с точностью 1).

### 5.4. get_grf(...)

Формирование графического отчёта по одному параметру:

- Построение графика (OxyPlot) по двум сериям: PSS введён / PSS выведен;
- Сохранение рисунка в PNG в папку режима;
- Вставка рисунка в документ Word с выравниванием и подписью (имя рисунка передаётся из вызывающего кода).

---

## 6. Опции пользователя

- **Сохранять графики?** (`grf_save`) — при включении вызывается `get_grf`, в Excel в «Примечание» пишется ссылка на рисунок, графики сохраняются в папку режима и в Word.
- **Сохранять режимы?** (`rst_save`) — сохранение файлов `.rst` нормальной и ремонтных схем в папку с именем базового режима.
- **Сохранять *.xls?** (`csv_save`) — при включении выполняется дополнительный экспорт в формате xls по режимам (логика в коде вокруг сохранения результатов).

## 7. Итоговая схема потока данных

```
Запуск → Main() → MainWindow
         ↓
    Window_Loaded: проверка/запись EPPlus, OxyPlot
         ↓
    Пользователь: Drag&Drop файлов (*.rst, *.ut2, *.sch, *.scn, *.xlsx)
         ↓
    compModel: разбор xlsx (rgms, rems, ut, arv, grf), сопоставление с путями файлов
         ↓
    Кнопка «Расчёт» → license_check → диалог «Начать расчет?»
         ↓
    Для каждого базового режима:
      → предварительная настройка com_dynamics и сохранение режима (до цикла уже сделано);
      → загрузка режима в Rastr;
      → rgm("") — сходимость?
         → нет: запись «Режим не балансируется в нормальной схеме», следующий режим;
         → да: get_pss_quality (DynCalc → расчёт с PSS вкл/выкл → оценка демпфирования и эффективности → при grf_save: get_grf);
      → для каждой ремонтной схемы режима:
           применение отключений ветвей/узлов → rgm("");
           при необходимости get_mpd (ограничение по МДП);
           get_pss_quality; при rst_save — сохранение .rst;
         ↓
    Сохранение «Приложение. Результат в табличном виде.xlsx» и «Приложение. Результат в графическом виде.docx»
         ↓
    Сообщение о завершении, разблокировка кнопки.
```
