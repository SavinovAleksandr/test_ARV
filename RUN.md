# Как запустить программу на другом компьютере

## Вариант «без .NET Framework» (автономная папка)

В проекте есть сборка под **.NET 8** с включённой в папку средой выполнения (self-contained). На целевом ПК с Windows **не нужно устанавливать .NET Framework**.

- **Готовая папка:** после выполнения в корне проекта команды публикации создаётся папка **`Publish_Standalone`** со всеми нужными файлами и `Модуль оценки АРВ.exe`. Скопируйте эту папку на другой компьютер с Windows и запустите exe.
- **Как собрать папку заново** (на машине с установленным .NET SDK):
  ```cmd
  dotnet publish avr_mu_computation_net8.csproj -c Release -r win-x64 --self-contained true -o Publish_Standalone
  ```
  Проект для этой сборки: **avr_mu_computation_net8.csproj**.

Подробнее про обычный запуск (с .NET Framework или из исходников) — ниже.

---

## 1. Требования к системе

- **ОС:** Windows (приложение WPF, запуск под macOS/Linux невозможен).
- **.NET Framework 4.7.2** — должен быть установлен ([скачать с сайта Microsoft](https://dotnet.microsoft.com/download/dotnet-framework/net472), если нет).
- Для **сборки из исходников** дополнительно: .NET SDK (например 6 или 8) или Visual Studio 2019+ с workload «.NET desktop development».
- Для **полной работы расчётов:** установленный **RUSTab** (ПК RUSTab, Astra) и **Microsoft Word** (для графического отчёта в .docx).

---

## 2. Вариант А: Запуск готового exe (если уже собран)

Если у вас есть папка с уже собранным приложением (например, скопирована с другой машины или из артефактов сборки):

1. Убедитесь, что установлен **.NET Framework 4.7.2**.
2. В одной папке должны лежать:
   - `Модуль оценки АРВ.exe`
   - `Модуль оценки АРВ.exe.config` (копия `app.config`)
   - `EPPlus.dll`, `OxyPlot.dll`, `OxyPlot.Wpf.dll`
   - при необходимости — `license.lic` (если включена проверка лицензии в коде).
3. Запуск: двойной щелчок по `Модуль оценки АРВ.exe` или из командной строки:
   ```cmd
   "Модуль оценки АРВ.exe"
   ```

Готовый набор файлов обычно лежит в каталоге сборки:  
`bin\Debug\net472\` или `bin\Release\net472\` (после сборки на Windows).

## 3. Вариант Б: Клонирование и сборка на другом ПК

### 3.1. Клонировать репозиторий

```cmd
git clone https://github.com/SavinovAleksandr/test_ARV.git
cd test_ARV
```

Или в папку с нужным именем:

```cmd
git clone https://github.com/SavinovAleksandr/test_ARV.git "модуль проверки АРВ"
cd "модуль проверки АРВ"
```

### 3.2. Сборка из командной строки

Нужен **.NET SDK** (не только Runtime). Установка: [.NET SDK](https://dotnet.microsoft.com/download).

```cmd
dotnet build avr_mu_computation.csproj -c Debug
```

Или Release:

```cmd
dotnet build avr_mu_computation.csproj -c Release
```

После успешной сборки exe будет в:

- Debug: `bin\Debug\net472\Модуль оценки АРВ.exe`
- Release: `bin\Release\net472\Модуль оценки АРВ.exe`

### 3.3. Запуск после сборки

Из командной строки из корня репозитория:

```cmd
bin\Debug\net472\Модуль оценки АРВ.exe
```

или перейти в `bin\Debug\net472\` и запустить exe оттуда.

### 3.4. Сборка и запуск из Visual Studio (Windows)

1. Установить Visual Studio 2019 или новее с компонентом «Разработка классических приложений .NET».
2. Открыть `avr_mu_computation.csproj` или `avr_mu_computation.sln`.
3. Выбрать конфигурацию Debug или Release, собрать (Ctrl+Shift+B).
4. Запуск: F5 или «Начать отладку».

## 4. Что должно быть на целевом компьютере для полной работы

| Компонент | Назначение |
|-----------|------------|
| .NET Framework 4.7.2 | Запуск приложения |
| RUSTab (ПК RUSTab, Astra) | Расчёты режимов, динамика, траектории (ASTRALib/Rastr) |
| Microsoft Word | Формирование «Приложение. Результат в графическом виде.docx» |
| Шаблоны Rastr | В папке пользователя: `%USERPROFILE%\Documents\RastrWin3\SHABLON\` (динамика.rst, сценарий.scn, автоматика.dfw, сечения.sch, траектория утяжеления.ut2 и т.д.) |

Без RUSTab приложение откроется, но при нажатии «Расчёт» возможны ошибки COM при создании Rastr. Без Word не получится сформировать графический отчёт в .docx (табличный .xlsx создаётся через EPPlus и не требует Word).

## 5. Частые проблемы

- **«Не удалось записать необходимые библиотеки»** — при первом запуске программа пытается записать EPPlus.dll, OxyPlot.dll, OxyPlot.Wpf.dll из ресурсов в свой каталог. Нужны права на запись в папку, где лежит exe. Либо положите эти DLL вручную в ту же папку, что и exe.
- **Ошибка при запуске расчёта (COM, Rastr)** — проверьте, установлен ли RUSTab и зарегистрирован ли COM-объект Astra.
- **Ошибка при создании документа Word** — установите Microsoft Word или отключите формирование графического отчёта в коде (только табличный xlsx).

Подробнее о сборке и зависимостях см. **BUILD.md**.
