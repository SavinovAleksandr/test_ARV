# Сверка кода с «Руководством пользователя»

Проверка соответствия восстановленного кода описанию в «Руководстве пользователя» («Модуль оценки параметров настройки АРВ», 2019).

---

## 1. Общее описание и назначение

**В руководстве:** Модуль выполняет оценку параметров настройки АРВ с использованием ПК RUSTab; формирование ремонтных схем; расчёт при введённых/выведенных каналах стабилизации; результаты в виде *.xlsx и *.docx.

**В коде:**
- `MainWindow.start_Click` — запуск расчётного цикла (проверка лицензии, затем цикл по режимам).
- `DynCalc(Rastr r)` — расчёты по Rastr (ASTRALib).
- `get_pss_quality`, `get_grf`, `get_mpd` — формирование результатов и работа с Excel/Word.
- Экспорт в xlsx (EPPlus) и в Word (Microsoft.Office.Interop.Word) реализован в коде.

**Вывод:** Соответствует.

---

## 2. Интерфейс и блоки

**В руководстве:**
- Расчётные режимы — файлы *.rst, двойной клик очищает.
- Траектории утяжеления — *.ut2, двойной клик очищает.
- Контролируемые сечения — один файл *.sch, двойной клик очищает.
- Расчётное задание — один *.xlsx.
- Тестовое возмущение — один *.scn.
- Установки: «Сохранять графики?», «Сохранять режимы?», «Сохранять *.xls?».
- Кнопка запуска расчёта.
- Индикатор выполнения расчётного цикла.
- Импорт перетаскиванием файлов на окно.

**В коде:**
- `files_drop` — обработка Drag&Drop; вызовы `rgms_add_to_main_win`, `ut2_add_to_main_win`, `sch_add_to_main_win`; обновление полей для scn и xlsx (`tvz`, `task`).
- `rgms_MouseDoubleClick`, `uts_MouseDoubleClick`, `schs_MouseDoubleClick` — очистка списков при двойном клике (в `IComponentConnector.Connect` подписаны обработчики).
- `compModel` — поля `regims`, `ut2s`, `schs`, `scns`, `xlxs` (соответствуют блокам и одному файлу задания).
- Чекбоксы `grf_save`, `rst_save`, `csv_save` — соответствуют «Сохранять графики?», «Сохранять режимы?», «Сохранять *.xls?».
- Кнопка `start` с обработчиком `start_Click`.
- `prBar` (ProgressBar) — индикатор; обновление через `updPrBar` и `RangeBase.ValueProperty` в цикле расчётов.

**Вывод:** Соответствует.

---

## 3. Необходимое ПО и библиотеки

**В руководстве:**
- Astra.dll (ПК RUSTab) с действующей лицензией.
- В корне программы: EPPlus.dll, OxyPlot.dll, OxyPlot.Wpf.dll. При отсутствии — автозапись из ресурсов; при ошибке записи — сообщение (рис. 4).

**В коде:**
- `ASTRALib` (Rastr, таблицы, Cols и т.д.) — используется во всех расчётах; создание через `Marshal.GetTypeFromCLSID` (RUSTab).
- `Window_Loaded`: проверка наличия EPPlus.dll, OxyPlot.dll, OxyPlot.Wpf.dll в `FileSystem.CurDir()`; при отсутствии — `File.WriteAllBytes` из `arv_mu_computation.My.Resources.Resources.EPPlus/OxyPlot/OxyPlot_Wpf`; при исключении — MsgBox с текстом про запись библиотек и контакты (И.Г. Выборных).

**Вывод:** Соответствует.

## 4. Лицензирование

**В руководстве:**
- Для запуска расчётного алгоритма нужен файл license.lic в корне.
- При нажатии «Расчёт» — проверка наличия и корректности license.lic. Возможны сообщения об отсутствии или некорректности (рис. 20, 21).

**В коде:**
- `license_check()` в `MainWindow`: в оригинале чтение `FileSystem.CurDir()\license.lic`, разбор символов по позициям строк, сравнение с именем компьютера; при ошибке — MsgBox и `Application.Current.Shutdown()`. В восстановленной сборке вызов `license_check()` оставлен, но тело метода заменено на пустое (обход лицензии).

**Вывод:** Логика лицензии в коде соответствует руководству; в текущей сборке проверка отключена по требованию.

---

## 5. Расчётный алгоритм

**В руководстве:**
- После нажатия «Расчёт» — проверка лицензии, затем диалог подтверждения (рис. 11).
- Последовательная обработка базовых режимов и ремонтных схем; проверка сходимости режима; при отсутствии сходимости — запись «Режим не балансируется…»; расчёт эффективности АРВ (введённые/выведенные каналы); при необходимости — ограничение перетоков по траектории утяжеления; сохранение результатов; обновление индикатора; по окончании — сообщение (рис. 12, 13).

**В коде:**
- `start_Click`: вызов `license_check()`, затем цикл по `fInf.xlxs.rgms`, для каждого режима — загрузка Rastr, сценария, шаблона; работа с таблицами `com_dynamics`, сохранение режимов/графиков/xls по флагам; при ошибках баланса — запись в ячейки «Режим не балансируется в нормальной схеме» / «Режим не балансируется в ремонтной схеме»; вызовы `get_pss_quality`, `DynCalc`, `get_mpd`, `get_grf`; обновление `prBar` через Dispatcher; в конце — сообщение об окончании расчётного цикла.

**Вывод:** Соответствует.

## 6. Формат результатов

**В руководстве:**
- Результаты в корне: «Приложение. Результат в графическом виде» (*.doc), «Приложение. Результат в табличном виде» (*.xlsx).
- Папки по наименованию базового режима — дополнительная информация.
- Табличный вид: «Схема сети», «Регистрируемый параметр», «Степень демпфирования», «Демпфирование переходного процесса», «Эффективность PSS», «Примечание».
- Графический вид: графики в *.png в папке режима, экспорт в Word; подписи под рисунками; при опции «Сохранять *.xls» — экспорт по режимам в *.xls, листы по регистрируемым параметрам, «PSS введен» / «PSS выведен».

**В коде:**
- Формирование xlsx через EPPlus: заголовки «Схема сети», «Регистрируемый параметр», «Степень демпфирования», «Результат проверки» (подзаголовки «Демпфирование переходного процесса», «Эффективность PSS»), «Примечание»; объединение ячеек, стили, цвета.
- Word: создание документа, вставка рисунков, выравнивание (WdParagraphAlignment), подписи.
- Сохранение по путям с именами режимов; экспорт xls при `csv_save`; графики (OxyPlot) и сохранение в папки/Word при `grf_save`.

**Вывод:** Соответствует.

## 7. Файл задания xlsx (листы rgms, rems, ut, arv, grf)

**В руководстве:**
- Листы «rgms», «rems», «ut», «arv», «grf» с описанными полями (номера, наименования, идентификаторы, МДП, сечения, траектории, таблицы/параметры RUSTab и т.д.).

**В коде:**
- `compModel.get_xlxs` и разбор xlsx: чтение листов по именам, заполнение `comInfo` (rgms, rems, ut, arv, grf); сопоставление имён файлов с полями; использование в циклах расчёта и при формировании ремонтных схем (rId, элементы, таблицы Rastr).

**Вывод:** Соответствует.

## Итог

Восстановленный код реализует описанные в руководстве:

- интерфейс (блоки, drag&drop, двойной клик, чекбоксы, кнопка «Расчёт», прогресс-бар);
- проверку и автозапись EPPlus/OxyPlot в корень;
- проверку license.lic (в сборке заменена на обход);
- расчётный цикл с RUSTab/ASTRALib, формирование xlsx и Word, сохранение режимов/графиков/xls по настройкам;
- форматы результатов и структуру файла задания xlsx.

Запуск полного функционала возможен только при установленном ПК RUSTab (Astra.dll) и при необходимости — наличии license.lic или оставленном обходе лицензии в коде.
